# セグメンテーション 実機検証チェックリスト（P0/P1/P2）

このセッションで実装した機能の**実機（描画・ペイント）確認**手順。型/ビルドは green だが、セグメンテーションは実機確認が必須。
関連: `fw/segmentation-tools-design.md`（設計）、`fw/roi-mask-progress.md`（実装状況）。

---

## 0. 事前準備

1. **起動**（standalone）: `bash scripts/dev-desktop.sh`（backend:8080 / Vite:5173 / Electron）。
2. **DevTools を開く**（Electron: Ctrl+Shift+I）。Console にセグメンテーション系の警告/例外が出ないか常時監視。
3. **テストデータ**:
   - **A) 規則的 CT ボリューム**（複数スライス・単一オリエンテーション・等間隔）… Brush/2D Wand/3D Wand/統計の主対象。
   - **B) 非ボリューム系**（localizer/scout・単一スライス・混在オリエンテーション・mosaic 等）… 3D Wand ガード（`isValidVolume`=false）の確認用。
4. 2D Viewer でシリーズをタイル表示（SliderView 単独タイル推奨。まず 1 タイルで確認）。
5. **ROI メニュー → Manager** で右パネル（ROI マネージャ）を開いておく。

> 凡例: 各項目 □=未確認 / ✅=OK / ❌=NG（NG は Console ログ＋再現手順をメモ）。

## G. 不具合時の切り分けメモ

- **Brush 起動が遅い（A1 NG）**: `/layout` レスポンスに `frameOfReferenceUID`・IOP・spacing・zSpatial が載っているか。無ければメタプロバイダが空 → 遅延ロードにフォール（=従来挙動）。backend 再ビルド要否を確認。
- **アクティブ対象に塗られない（B2/B4 NG）**: Console で `getSegEditTarget()` の状態、`ensureStackSegmentation`／`activateMask` のログ。◉ の選択と Cornerstone の active segmentation がズレていないか。
- **2D Wand が塗れない/広がりすぎ（D NG）**: シード画素の輝度（raw 値）とトレランス単位（**raw 画素値**、rescale 前）の関係。トレランスを大小に振って挙動確認。
- **3D Wand が例外（E1 NG）**: Console に "Volume is not reconstructable" 系。`isValidVolume` が true か（規則的シリーズか）。斜位ビューは非対応。
- **色が変わらない/全 segment 変わる（B8 NG）**: `setSegmentIndexColor` の対象 index（アクティブ segment）と表示中ビューポート。
